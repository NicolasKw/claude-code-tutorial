**1. Add completedAt logic to `src/app/actions/progress.ts`:**

Before the upsert, check if all levels are complete and `completedAt` is not yet set. If so, add it to the data object:

```typescript
import { TOTAL_LEVELS } from '@/lib/types/tutorial';

// Inside saveProgress, before the upsert:
if (
  data.completedLevels.length >= TOTAL_LEVELS &&
  !data.completedAt
) {
  data = { ...data, completedAt: new Date().toISOString() };
}
```

This ensures `completedAt` is set exactly once (the `!data.completedAt` guard prevents overwriting on subsequent saves).

**2. Update LevelPage navigation for final level:**

- Change the AchievementOverlay `onNavigate` callback:

Replace:
```tsx
<AchievementOverlay
  show={showOverlay}
  level={level}
  summary={levelData?.summary}
  onNavigate={() => router.push(`/tutorial/${level + 1}`)}
```

With:
```tsx
<AchievementOverlay
  show={showOverlay}
  level={level}
  summary={levelData?.summary}
  onNavigate={() => {
    if (level >= TOTAL_LEVELS - 1) {
      const userId = getUserId();
      router.push(userId ? `/certificate/${userId}` : '/');
    } else {
      router.push(`/tutorial/${level + 1}`);
    }
  }}
/>
```

This reads `userId` from localStorage (set during registration by Plan 01 changes). If userId is somehow missing, falls back to landing page.
