## Interaction Contract

### Download Button ("Descargar certificado")

1. User clicks button
2. Button enters loading state: spinner replaces `Download` icon, text "Descargando...", pointer-events none
3. `fetch(/api/certificate/[userId])` → blob → `URL.createObjectURL` → `<a download="certificado-claude-code.png">` programmatic click
4. Button returns to default state after 500ms delay (gives browser time to trigger download)
5. On fetch error: button returns to default state, error toast appears (see Copywriting Contract)

States:
- Default: bg `#D9ADFF`, text `#171717`, weight 600, `Download` icon
- Loading: opacity 70%, spinner icon, pointer-events none
- Error: returns to default (error handled by toast)

### LinkedIn Share Button ("Compartir en LinkedIn")

1. User clicks button
2. `window.open('https://www.linkedin.com/sharing/share-offsite/?url={encodedCertUrl}', '_blank', 'noopener,noreferrer,width=600,height=600')`
3. Button has no loading state — fire and forget

States:
- Default: bg `#70B5FF`, text `#171717`, weight 600, `Linkedin` icon
- Hover: opacity 80%
- Active: scale 97%

### Copy Post Text Button ("Copiar texto")

1. User clicks button
2. `navigator.clipboard.writeText(postText)`
3. Button enters "copied" state for 2 seconds: `Check` icon replaces `Copy` icon, text "Copiado"
4. Returns to default after 2s timeout
5. On clipboard error: silent fail (no UI change — clipboard unavailable means user can select text manually)

States:
- Default: variant outline, `#D9ADFF` border/text, `Copy` icon
- Copied (2s): `#D9ADFF` border, `Check` icon, text "Copiado"

### Certificate Preview Image

- Renders as `<img>` with `src="/api/certificate/[userId]"`, width 100%, aspect ratio forced to 1200/630 via `style="aspect-ratio: 1200 / 630"`
- Shows a `rounded-lg` loading skeleton (Card background color) while the image loads (use `onLoad` to swap)
- On 404 (userId not found): show error state instead of broken image

## Implementation Notes for Executor

1. **Certificate page is public.** Do NOT wrap `/certificate/[userId]` in `SessionGuard`. The route must be accessible to unauthenticated users and LinkedIn's crawler. (Pitfall 5 in 04-RESEARCH.md.)

2. **Satori CSS subset.** Use only `display: 'flex'` inside ImageResponse JSX. Do not use `display: 'grid'`, `gap` shorthand, or `var(--brand-primary)` — pass literal hex values. (Pitfall 2 in 04-RESEARCH.md.)

3. **TTF font required.** Add Inter-Regular.ttf and Inter-Bold.ttf to `public/fonts/`. Load with `fs.readFile(join(process.cwd(), 'public/fonts/Inter-Bold.ttf'))` inside the Route Handler. (Pitfall 3 in 04-RESEARCH.md.)

4. **userId in localStorage.** Store `userId` alongside `sessionId` at registration time (one-line change in `RegistrationForm.tsx` — `registerUser` action already returns the userId). Read from localStorage in `LevelPage.tsx` for the "Ver mi certificado" navigation. (Pitfall 4 in 04-RESEARCH.md.)

5. **completedAt timestamp.** Add `completedAt?: string` to `TutorialProgress` type (ISO string). Set it once when `completedLevels.length === TOTAL_LEVELS` in the `saveProgress` action. No DB migration required — stored in existing JSONB `data` column. (Pitfall 6 in 04-RESEARCH.md.)

6. **Download pattern.** Use `fetch` + `URL.createObjectURL` + `<a download>` click. Do NOT use `window.open(imageUrl)` — that opens in a new tab. (Anti-pattern noted in 04-RESEARCH.md.)

7. **LinkedIn share URL.** Use `https://www.linkedin.com/sharing/share-offsite/?url={encodedUrl}`. Do NOT use the legacy `shareArticle?mini=true` endpoint. (04-RESEARCH.md Pattern 4.)

8. **NEXT_PUBLIC_BASE_URL env var.** Required for absolute `og:image` URL. Must be set in `.env.local` (development) and Vercel environment variables (production).
