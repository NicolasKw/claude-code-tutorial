## Layout Contract

### Global Tutorial Layout

Every `/tutorial/[level]` page uses a two-zone layout:

```
┌─────────────────────────────────────────────────────┐
│  STICKY HEADER (h-14, z-50, bg-foreground)          │
│  [ZalesMachine logo]  [Nivel 3 de 7 badge] [====]   │
│                        badge + full-width progress  │
└─────────────────────────────────────────────────────┘
│                                                     │
│  CONTENT AREA (pt-14 to clear header)               │
│  max-w-2xl mx-auto px-4                             │
│  Level heading, steps, code blocks                  │
│                                                     │
└─────────────────────────────────────────────────────┘
```

### Progress Header (TUTO-06)

- Position: `sticky top-0 z-50`
- Background: `#171717` (foreground color)
- Height: `h-14` (56px)
- Contents:
  - Left: ZalesMachine wordmark (white, 14px, weight 600)
  - Center: Badge — "Nivel X de 7" using shadcn Badge component, variant `secondary`
  - Bottom edge: shadcn Progress component, full width, `h-2`, fill color `#D9ADFF`
- Progress value: `(currentLevel / 7) * 100`

### Level Page Content

- Container: `max-w-2xl mx-auto px-4 py-8`
- Level title: `<h1>` at 20px / weight 600 / line-height 1.2
- Step list: ordered, each step separated by `lg` (24px) vertical gap
- "Listo, ya lo hice" button: full-width on mobile, `max-w-xs` on desktop, 44px min-height, bg `#D9ADFF`, text `#171717`, weight 600

### Locked Level State (TUTO-01)

When a user navigates to a locked level URL:
- Show the level title (grayed out, opacity 40%)
- Show a lock icon (lucide `Lock`, 24px) centered on the content area
- Copy: "Este nivel está bloqueado" (heading 20px), "Completá el Nivel X para desbloquearlo." (body 16px)
- Show "Volver al Nivel X" button (shadcn Button, variant `outline`)
- No progress bar advancement

### Code Block Component (TUTO-05)

- Background: `#171717` (dark surface)
- Font: `var(--font-geist-mono)`, 14px, weight 400, line-height 1.6
- Padding: `md` (16px) all sides
- Border radius: 8px
- Copy button: top-right corner, lucide `Copy` icon (16px), switches to lucide `Check` (16px) for 2 seconds after click
- Copy button color: `#D9ADFF` on dark background
- Prose-to-code distinction: code blocks have dark background + mono font; prose has white background + sans font — no additional border needed

## Achievement Overlay (TUTO-02)

Triggered when user clicks "Listo, ya lo hice" on the last step of a level.

Layout: full-screen modal (`fixed inset-0 z-50 bg-black/60 backdrop-blur-sm`), content centered.

Card (shadcn Card, max-w-sm, centered):
- Background: `#E9FFB9` (success/lime green)
- Padding: `lg` (24px)
- Structure:
  1. Heading: "Nivel X completado" — 28px / weight 600 / line-height 1.1 / color `#171717`
  2. Subheading: "Qué lograste" — 14px / weight 600 / color `#171717` / margin-top `md` (16px)
  3. Summary body: placeholder prose (Phase 3 fills content) — 16px / weight 400 / line-height 1.5 / color `#171717`
  4. CTA button: "Ir al Nivel X+1" — shadcn Button, full-width, bg `#70B5FF`, text `#171717`, weight 600, 44px height — margin-top `lg` (24px)

Special case — final level (Level 7 completion):
- CTA button copy: "Ver mi certificado" (Phase 4 wires destination)
- Button bg: `#D9ADFF`

Dismiss behavior: clicking CTA closes overlay AND navigates to next level URL. No click-outside-to-dismiss (prevents accidental dismissal).

## Step Completion Flow (TUTO-03)

Each step within a level:

1. Step body rendered (prose + optional code block)
2. "Listo, ya lo hice" button at the bottom of the step
3. On click: step marked complete in state, next step reveals with a fade-in (150ms ease-out)
4. Final step click triggers Achievement Overlay

Button states:
- Default: bg `#D9ADFF`, text `#171717`, weight 600
- Completed: bg transparent, border 1px `#D9ADFF`, text `#D9ADFF`, lucide `Check` icon inline-left, pointer-events none
- Disabled (future steps): hidden until previous step complete

## Accessibility Contract

- All interactive elements: minimum 44px touch target
- Progress bar: `role="progressbar"` with `aria-valuenow`, `aria-valuemin="0"`, `aria-valuemax="100"`, `aria-label="Progreso del tutorial"`
- Achievement modal: `role="dialog"`, `aria-modal="true"`, focus trap on open, focus returns to level page on close
- Code copy button: `aria-label="Copiar código"` / `aria-label="Código copiado"` toggle
- Locked level icon: `aria-hidden="true"` (decorative)
- Color contrast: all text on `#E9FFB9` background verified ≥ 4.5:1 (`#171717` on `#E9FFB9` = ~13:1, passes)

## Implementation Notes for Executor

1. **shadcn init first.** Run `npx shadcn@latest init` before any component work. Select New York style. This auto-detects Tailwind v4 and creates `components.json`.

2. **Brand tokens in globals.css.** Add `--brand-primary`, `--brand-secondary`, `--brand-success` to the existing `:root` block. Do not remove existing tokens.

3. **Progress persistence.** Phase 2 uses localStorage (matching Phase 1's session pattern). Key: `tutorial_progress`. Shape: `{ currentLevel: number, completedLevels: number[], completedSteps: Record<number, number[]> }`.

4. **Phase 3 placeholder content.** Each level page needs a content slot. Use a `children` or `content` prop so Phase 3 can drop in the real copy without touching the shell.

5. **Tailwind v4 note.** No `tailwind.config.*` file exists — Tailwind v4 uses CSS-first config via `@theme` in `globals.css`. Add brand color tokens there, not in a JS config file.

6. **Dark mode.** Dark mode is explicitly out of scope (v2+). The existing `prefers-color-scheme` dark block in `globals.css` can be left as-is but should not be extended for Phase 2 components.
